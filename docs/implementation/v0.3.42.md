## Summary

PLAN_FOR_V0_3_0 residual polish: **template secret inference** for Media/Deploy
(Tasks 7–8), **media size/voice + deploy project name + Discord length**
validation, plus **v0.3.41 review fixes** (message-text fallthrough, Slack

## Changes

### Task 1: Template required settings (Tasks 7–8)

- **`inferRequiredSettings`** also maps:
  - `media` → `OPENAI_API_KEY`
  - `deploy` → `VERCEL_API_TOKEN` (default) or Cloudflare token + account id

### Task 2: Media / deploy / Discord graph validation

- Shared **`media-node-options.ts`**: image sizes, TTS voices, Discord max length,
  deploy project name regex
- **`invalid_media_size`** / **`invalid_media_voice`** when config is set but not
  in the NodeConfig allow-list
- **`invalid_deploy_project`** for illegal project names
- **`discord_content_too_long`** when static content exceeds 2000 chars

### Task 3: v0.3.41 review fixes (carried forward)

- message-text: whitespace-only `textTemplate` falls through to `content`/`text`
- Slack: trim channel; treat API `ok: false` as failure
- Deployments: stale workflow filter cleared in a dedicated effect; trim ids

### Task 4: Unit tests

- template-required-settings media/deploy cases
- media-node-options helpers
- WorkflowValidation new codes
- expanded message-text / Slack / deployment-filter tests from review
