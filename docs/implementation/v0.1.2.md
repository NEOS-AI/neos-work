# NEOS Work v0.1.2 — 구현 내역

> **기준 버전**: v0.1.1
> **릴리즈일**: 2026-04-01
> **계획 문서**: [docs/PLAN_FOR_V0_1_2.md](./PLAN_FOR_V0_1_2.md)

## 개요

v0.1.2는 에이전트 파이프라인을 HTTP SSE로 노출하고(Tier 1), 웹 검색·셸·파일 검색 툴을 추가하며(Tier 2), MCP 클라이언트 연동(Tier 3), 영구 메모리 시스템(Tier 4), 스킬 시스템(Tier 5)을 구현한 기능 확장 릴리즈다.

## Tier 1 — 에이전트 HTTP SSE 라우트 (A5/A6)

### A5: `POST /api/session/:id/agent`

`AgentOrchestrator`를 Server-Sent Events로 노출하는 엔드포인트.

**SSE 이벤트 목록:**

| 이벤트 | 데이터 | 설명 |
|--------|--------|------|
| `plan` | `{ steps: AgentStep[] }` | 플래너가 생성한 전체 단계 목록 |
| `step_start` | `{ step: AgentStep }` | 단계 실행 시작 |
| `step_complete` | `{ step: AgentStep }` | 단계 성공 완료 |
| `step_error` | `{ step: AgentStep, error: string }` | 단계 실패 (비중단, 다음 단계 계속) |
| `text` | `{ content: string }` | 합성 응답 스트리밍 |
| `done` | `{ task: AgentTask }` | 전체 완료 |
| `error` | `{ error: string }` | 치명적 오류 |

**취소 처리:**
- 기존 `activeChats` Map 재활용 — `AbortController` 저장
- `POST /api/session/:id/cancel` 호출 시 수동 취소 신호 발생
- 클라이언트 연결 해제 시 자동 취소
- `AbortSignal.any([clientSignal, cancelSignal])`로 두 신호를 병합

**DB 연동:**
- `stepDbIds` Map으로 내부 step ID → DB `agent_step` 행 ID 매핑
- `step_start`/`step_complete`/`step_error`마다 `agent-steps.ts` CRUD로 상태 기록

**툴 확인 플로우 (VULN-003):**
- `write_file`, `run_command` 등 파괴적 툴은 실행 전 사용자 승인 필요
- `pendingConfirmations` Map에 툴 사용 ID와 Promise resolver 저장
- `POST /api/session/:id/tool-confirm/:toolUseId` 호출로 approve/reject
- 60초 미응답 시 자동 reject

**메모리 콜백 주입:**
- 세션 시작 시 `MemoryCallbacks` 객체 생성 (워크스페이스 스코프)
- `createRememberTool()`, `createRecallTool()`, `createForgetTool()` 에 콜백 전달
- `loadMcpTools()`로 활성화된 MCP 서버 툴도 동적 등록

**구현 파일:** `apps/server/src/routes/session.ts`

### A6: 에이전트 실행 UI

**`engine.ts` — `runAgent()` 메서드:**
```typescript
async *runAgent(sessionId: string, content: string, signal?: AbortSignal): AsyncGenerator<AgentChunk>
```
- `/api/session/:id/agent`로 POST 요청
- SSE 스트림을 파싱해 타입이 지정된 `AgentChunk` discriminated union으로 yield

**`AgentChunk` 타입:**
```typescript
type AgentChunk =
  | { type: 'plan'; steps: AgentStep[] }
  | { type: 'step_start'; step: AgentStep }
  | { type: 'step_complete'; step: AgentStep }
  | { type: 'step_error'; step: AgentStep; error: string }
  | { type: 'text'; content: string }
  | { type: 'done'; task: AgentTask }
  | { type: 'error'; error: string }
```

**`Sessions.tsx` — `AgentPlanCard` 컴포넌트 (라인 854-927):**
- 플랜 단계 목록 — 접기/펼치기 가능
- 진행 카운터: `{completedCount}/{plan.length} steps`
- 스트리밍 중 애니메이션 펄스 인디케이터
- 단계별 상태 아이콘:
  - ○ pending (빈 원)
  - ⟳ running (스피너)
  - ✓ completed (체크마크)
  - ✗ error (X 표시)
- 실패 단계에 에러 메시지 인라인 표시

**`isAgentMode` 토글:**
- 입력 푸터에 모드 전환 버튼
- `handleSendAgent()` → `client.runAgent()` 라우팅

**구현 파일:** `apps/desktop/src/lib/engine.ts`, `apps/desktop/src/pages/Sessions.tsx`

## Tier 2 — 툴 확장 (T1/T2/T3)

### T1: `web_search`

**구현 파일:** `packages/core/src/tools/web-search.ts`

```typescript
// Input
{ query: string; maxResults?: number }  // maxResults: 1-10, 기본값 5
// Output
{ results: { title: string; url: string; snippet: string }[] }
```

- **백엔드**: DuckDuckGo Instant Answer API (인증 키 불필요)
- **파라미터**: `format=json`, `no_html=1`, `skip_disambig=1`
- **파싱**: Abstract(최상위 결과) + RelatedTopics(추가 결과)
- **타임아웃**: `AbortSignal.timeout(10_000)` — 10초

**SSRF 방어 (`isPrivateHost()`):**

차단 대상 패턴:
- IPv4: `127.*`, `10.*`, `172.16-31.*`, `192.168.*`
- IPv6: `::1`, `fc00:`, `fe80:`
- 로컬호스트: `localhost`

패치 URL 생성 전 호스트명 검증 — 내부 네트워크 접근 차단.

### T2: `run_command`

**구현 파일:** `packages/core/src/tools/shell.ts`

```typescript
// Input
{ command: string; cwd?: string; timeout?: number }
// Output
{ stdout: string; stderr: string; exitCode: number; note?: string }
```

**제한 사항:**
- 기본 타임아웃: 30초 / 최대: 120초

**금지 명령 패턴 (정규식):**

| 카테고리 | 패턴 |
|----------|------|
| 파일 삭제 | `rm -rf` (경로 포함) |
| 권한 상승 | `sudo`, `su` |
| 권한 변경 | `chmod 777` |
| 디스크 조작 | `dd`, `mkfs`, `fdisk`, `format` |
| 네트워크 설정 | `ifconfig`, `ip`, `iptables` |
| 서비스 제어 | `systemctl`, `launchctl` |
| 파이프 실행 | `curl\|bash`, `wget\|bash` |

**CWD 샌드박스:**
- `realpathSync()`로 실제 경로 확인 (심볼릭 링크 탈출 방지)
- workspace 루트 외부 경로 거부
- AbortSignal 연동 → SIGTERM 전송

### T3: `search_files` + `move_file`

**구현 파일:** `packages/core/src/tools/filesystem.ts` (기존 파일 확장)

```typescript
// Input
{ pattern: string; directory?: string; type?: 'glob' | 'content' }
```

```typescript
// Input
{ source: string; destination: string }
// Output
```

- 파일 및 디렉터리 이동·이름 변경 지원

**공통 보안 (`safePath()`):**
- `..` 경로 순회 차단

## Tier 3 — MCP 클라이언트

### `packages/mcp-client/`

**구성 파일:**
- `client.ts` — MCP 서버 연결 및 툴 실행
- `tool-bridge.ts` — MCP 툴 → core `Tool` 인터페이스 변환
- `discovery.ts` — 설정 기반 서버 목록 로드
- `index.ts` — 패키지 진입점

**`McpClient` 인터페이스:**

| 메서드 | 설명 |
|--------|------|
| `connect(config)` | 서버 연결 (stdio 또는 http) |
| `listTools()` | 사용 가능한 툴 목록 반환 |
| `callTool(name, input)` | 툴 실행, `{success, output}` 반환 |
| `disconnect()` | 연결 종료 |
| `connected` | 연결 상태 getter |

**전송 방식:**
- `stdio`: `StdioClientTransport` — 서브프로세스 스폰
  - `command`: 실행 명령 (예: `npx`)
  - `args`: 인수 배열 (예: `["-y", "@modelcontextprotocol/server-filesystem", "/path"]`)
- `http`: `SSEClientTransport` — HTTP SSE
  - `url`: 서버 URL (예: `http://localhost:3000/sse`)

**`tool-bridge.ts`:**
- `mcpToolToTool()` — MCP `McpToolDefinition` → core `Tool` 타입 변환
- `buildMcpTools(client)` — 클라이언트의 모든 툴을 core Tool 배열로 빌드

**세션 통합 (`loadMcpTools()`):**
- `mcp_server` 테이블에서 `enabled=1` 서버 조회
- 각 서버에 `McpClient.connect()` 호출
- 성공 시 `ToolRegistry`에 툴 등록
- 연결 실패 시 에러 로그만 남기고 세션 계속 진행 (graceful)
- 채팅 및 에이전트 세션 시작 시 모두 호출

### API 엔드포인트 (`apps/server/src/routes/mcp.ts`)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/api/mcp-servers` | 전체 목록 |
| `POST` | `/api/mcp-servers` | 신규 서버 추가 |
| `POST` | `/api/mcp-servers/:id/toggle` | 활성화/비활성화 |
| `DELETE` | `/api/mcp-servers/:id` | 서버 삭제 |

### Settings UI (`apps/desktop/src/pages/Settings.tsx` — `McpServersSection`)

- 서버 목록 (활성화 토글, 삭제 버튼)
- 추가 폼: 전송 방식 선택 (stdio / http)
  - stdio: command 입력 + args (공백 구분) 입력
  - http: URL 입력
- 마운트 시 및 추가/토글/삭제 후 목록 자동 갱신

## Tier 4 — 메모리 시스템

### DB CRUD (`apps/server/src/db/memory.ts`)

| 함수 | 설명 |
|------|------|
| `createMemory(params)` | `ON CONFLICT (workspace_id, key) DO UPDATE` upsert |
| `getMemory(workspaceId, key)` | 단일 메모리 조회 |
| `searchMemory(workspaceId, query, tags?, limit=10)` | 전문 검색 + 태그 필터 |
| `listMemories(workspaceId, limit=20)` | 최신순 목록 |
| `deleteMemory(workspaceId, key)` | 삭제 |

- **워크스페이스 격리**: 모든 쿼리에 `workspace_id` 조건 포함
- **태그 필터**: JSON 배열 파싱 후 `tags.some()` — 하나라도 일치하면 반환

### 메모리 툴 (`packages/core/src/tools/memory.ts`)

```typescript
// remember
{ key: string; content: string; tags?: string[] }  → { success: boolean }

// recall
{ query: string; tags?: string[]; limit?: number }  → { memories: { key, content, tags }[] }

// forget
{ key: string }  → { success: boolean }
```

**`MemoryCallbacks` 인터페이스 (콜백 패턴):**
- `core` 패키지가 DB(`better-sqlite3`) 의존성 없이 작동
- 툴 생성 함수: `createRememberTool(cb)`, `createRecallTool(cb)`, `createForgetTool(cb)`
- `apps/server/src/routes/session.ts`에서 워크스페이스 스코프 콜백 생성 후 주입

**유효성 검사:**
- `key`: 1-200자
- `content`: 1-10,000자
- `tags`: 선택적 문자열 배열

### 신규 테이블

**`agent_step`**
```sql
CREATE TABLE IF NOT EXISTS agent_step (
  id          TEXT PRIMARY KEY,
  session_id  TEXT NOT NULL REFERENCES session(id) ON DELETE CASCADE,
  step_index  INTEGER NOT NULL,
  type        TEXT NOT NULL,
  status      TEXT NOT NULL,  -- 'pending' | 'running' | 'completed' | 'error'

  error       TEXT,
  created_at  TEXT DEFAULT (datetime('now')),
  updated_at  TEXT DEFAULT (datetime('now'))
```

```sql
CREATE TABLE IF NOT EXISTS memory (
  id           TEXT PRIMARY KEY,
  workspace_id TEXT NOT NULL REFERENCES workspace(id) ON DELETE CASCADE,
  key          TEXT NOT NULL,
  content      TEXT NOT NULL,
  tags         TEXT,           -- JSON array
  created_at   TEXT DEFAULT (datetime('now')),
  updated_at   TEXT DEFAULT (datetime('now')),
  UNIQUE(workspace_id, key)
);
```

```sql
CREATE TABLE IF NOT EXISTS skill (
  id           TEXT PRIMARY KEY,
  name         TEXT NOT NULL UNIQUE,
  description  TEXT,
  source       TEXT NOT NULL,  -- 'local' | 'global'
  path         TEXT NOT NULL,
  version      TEXT,
  enabled      INTEGER NOT NULL DEFAULT 1,
  installed_at TEXT DEFAULT (datetime('now'))
);
```

```sql
CREATE TABLE IF NOT EXISTS mcp_server (
  id          TEXT PRIMARY KEY,
  name        TEXT NOT NULL,
  transport   TEXT NOT NULL,  -- 'stdio' | 'http'

  args        TEXT,           -- JSON array
  url         TEXT,           -- http 전용
  enabled     INTEGER NOT NULL DEFAULT 1,
  created_at  TEXT DEFAULT (datetime('now'))
);
```

## 아키텍처 노트

### 에이전트 실행 플로우

```
사용자 입력
  → POST /api/session/:id/agent
    → loadMcpTools() + createMemoryTools()  # 세션별 툴 동적 등록
    → Planner.plan(goal)                    # LLM이 단계 분해, JSON 파싱
      → SSE: plan { steps }
    → AgentOrchestrator 루프
      → 각 단계:
          SSE: step_start
          executeStep(step)
            - toolName 있음: 직접 실행
            - toolName 없음: LLM에 툴 선택 위임
          SSE: step_complete | step_error
    → Synthesis: LLM이 완료 단계 결과 합성
      → SSE: text (스트리밍)
    → SSE: done
```

### 주요 설계 결정

| 결정 | 이유 |
|------|------|
| **콜백 패턴** (`MemoryCallbacks`) | `core` 패키지가 `better-sqlite3`에 의존하지 않도록 분리. 테스트 용이성 향상 |
| **MCP lazy loading** | 세션 시작 시 연결 — 서버 전체 시작 의존성 없음. 실패해도 세션 미중단 |
| **SSE 아키텍처** | 채팅·에이전트 모두 동일 패턴. 클라이언트 측 처리 일관성 |
| **VULN-003 (파괴적 툴 승인)** | `write_file`, `run_command` 실행 전 사용자 확인. 60초 타임아웃으로 데드락 방지 |
| **`AbortSignal.any()`** | 클라이언트 연결 해제 + 수동 취소 두 신호를 하나로 병합 |
| **워크스페이스 스코프 메모리** | `workspace_id` FK로 메모리 격리 — 워크스페이스 간 데이터 누수 방지 |
