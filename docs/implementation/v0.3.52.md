# v0.3.52 Implementation

## Summary

PLAN_FOR_V0_3_0 residual polish after the v0.3.51 review batch: **encrypt media/deploy
secrets at rest** (`isSensitiveKey` coverage for OpenAI / Vercel / Cloudflare), plus
**settings/media test hygiene** so parallel suites do not race on shared secret keys.

기준: v0.3.51 (including post-ship review: secret trim, coding bounds, Agent/Block hygiene)

---

## Changes

### Task 1: Encrypt media/deploy secrets (Tasks 7–8)

Previously only Anthropic/Google/Tavily/Slack/Discord/KIS keys were marked sensitive.
Media (`OPENAI_API_KEY`) and Deploy (`VERCEL_API_TOKEN`, `CLOUDFLARE_API_TOKEN`,
`CLOUDFLARE_ACCOUNT_ID`) were stored and listed in plain text.

- **`isSensitiveKey`** now includes:
  - `OPENAI_API_KEY`
  - `VERCEL_API_TOKEN`
  - `CLOUDFLARE_API_TOKEN`
  - `CLOUDFLARE_ACCOUNT_ID`
- Existing plaintext rows are re-encrypted on startup via `migrateEncryption()`
- Non-secret URLs (`OPENAI_BASE_URL`, etc.) remain non-sensitive

### Task 2: Test hygiene

- **settings-execution**: prefer KIS/Tavily keys for trim assertions (avoids races
  with media/deploy route suites sharing OPENAI/VERCEL settings)
- **media routes**: drop duplicate whitespace-prompt test case
- Crypto unit tests assert the new sensitive keys

### Task 3: Version

- Monorepo packages + server health/`index` version → **0.3.52**
