# v0.3.53 Implementation

## Summary

PLAN_FOR_V0_3_0 residual polish: **Media generator option clamps** (Task 7),
**Deploy project name validation** (Task 8), **secret write-trim** (Tasks 3/7–8),
**Design System id/content hygiene** (Task 1), and **Webhook signature header
trim** (Task 13).

기준: v0.3.52

---

## Changes

### Task 1: Media generator clamps (Task 7)

- **`generateImage`**: trim prompt (reject blank); clamp invalid `size` →
  `1024x1024`, invalid `quality` → `standard`
- **`generateAudio`**: trim text (reject blank); clamp invalid `voice` →
  `alloy`, invalid `model` → `tts-1`
- Allow-lists exported: `IMAGE_SIZES`, `IMAGE_QUALITIES`, `TTS_VOICES`, `TTS_MODELS`

### Task 2: Deploy project name validation (Task 8)

- **`isValidDeployProjectName`** on server (same regex as desktop graph validation)
- Deploy **POST** rejects invalid names with 400
- Deploy **preflight** marks `projectName` check failed when invalid

### Task 3: Secret write-trim (Tasks 3 / 7–8)

- **`setSetting`**: trim sensitive values before encrypt
- Empty-after-trim secrets store as plain `""` (encrypting empty broke `isEncrypted`)

### Task 4: Design System hygiene (Task 1)

- **`getDesignSystem` / `getDesignSystemContent`**: trim id; whitespace-only id → null
- Whitespace-only `DESIGN.md` treated as missing content
- **`updateWorkflow`**: trim `designSystemId`; blank → null

### Task 5: Webhook signature header (Task 13)

- **`verifyWebhookSignature`**: trim header and signature hex so padded
  `X-Neos-Signature` still verifies

### Task 6: Unit tests

- Media generator clamp / blank cases
- Deploy project name unit + route cases
- Design system id/content whitespace
- Webhook signature trim
- setSetting sensitive trim + non-sensitive padding preserved

### Review fixes (post-ship)

- **`isValidDeployProjectName`** shared in `@neos-work/shared` (desktop/server/DeployNode)
- **DeployNode**: reject invalid project name before fetch
- **MediaNode**: normalize `mediaType` case/whitespace
- **media-generator**: trim `apiKey`; reject blank
- **Workflow create/update**: trim name/description; reject blank name
- **Routines create/update**: trim name/workflowId/schedule/timezone
- **Design Systems create**: trim name/description
- **Settings verify-key**: trim provider/key; normalize provider case
- **Webhook**: trim secret before HMAC
- Extra coverage: executor, browser-tool, memory tools
