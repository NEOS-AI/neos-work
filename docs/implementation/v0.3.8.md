# v0.3.8 Implementation

## Summary

PLAN_FOR_V0_3_0 잔여 공백: CLI spawn `NEOS_*` env + per-run workspace,
revision **POST restore** API, artifact **preview** endpoint, ZIP에 DESIGN.md 포함,
GenUI **Confirmation** 컴포넌트 분리.

기준: v0.3.7

---

## Changes

### Task 1: CLI context env + workspace (PLAN Task 3)

- **`spawnCliAgent`**
  - `NEOS_SERVER_URL`, `NEOS_AUTH_TOKEN`, `NEOS_WORKFLOW_ID`, `NEOS_RUN_ID` 주입
  - `runId` 있으면 `~/.config/neos-work/workspaces/<runId>/` cwd 생성
- **`runtime-context`** — 서버 기동 시 auth token / port 공유
- workflow / webhook / routine-scheduler 실행 경로 연동

### Task 2: Revision restore API (PLAN Task 16)

- **`POST /api/workflow-revisions/:workflowId/:id/restore`**
  - snapshot → workflow 레코드 적용
  - 적용 전 현재 상태 `pre-restore` 스냅샷
- **`EngineClient.restoreRevision`**

### Task 3: Artifact preview endpoint (PLAN Task 4)

- **`GET /api/artifacts/:id/preview`** — raw HTML/text + CSP frame-ancestors

### Task 4: ZIP export DESIGN.md (PLAN Tasks 1 / 10)

- `designSystemId` 있으면 `design-systems/<id>/DESIGN.md` (+ meta) 포함

### Task 5: GenUIConfirmation (PLAN Task 6)

- **`GenUIConfirmation.tsx`** 분리
- **PipelineRunner** confirmation surface 연동

### Misc

- HTML artifact auto-detect DOCTYPE / `<html` 명시 처리
