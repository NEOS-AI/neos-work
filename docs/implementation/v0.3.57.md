# v0.3.57 Implementation

## Summary

PLAN_FOR_V0_3_0 residual polish: **Artifact create/update hygiene** (Task 4),
**Workspace path/name trim** (path-safety), and **Plugin LLM stage network
hygiene** (Task 5) so network/API failures return stage placeholders instead of
throwing.

## Changes

### Task 1: Artifacts (Task 4 Live Artifact)

- **POST**: trim `workflowId`, `name`, `contentType`, `runId`, `nodeId`; reject
  blank required fields; name max 200
- **PUT content**: reject whitespace-only content
- **PATCH**: trim name; reject blank/whitespace-only content

### Task 2: Workspaces + path safety

- **`validateWorkspacePath`**: trim before resolve; blank → false
- Workspace **POST/PUT**: trim name/path/type; reject blank name or invalid path

### Task 3: Plugin runner LLM stages (Task 5)

- Trim API keys before use; whitespace key → missing-key placeholder
- Wrap Anthropic/OpenAI `fetch` in try/catch
- Non-OK HTTP → stage error string (`API error N`)
- Network errors → stage error string (pipeline continues)

### Task 4: Unit tests

- Artifacts trim/blank/whitespace content cases
- Path-safety trim cases
- Workspace create trim
- Plugin runner network failure does not throw

### Review fixes (post-ship)

- **Plugin LLM keys**: trim before provider selection; whitespace Anthropic no
  longer blocks OpenAI fallback
- **Artifacts**: trim list query `workflowId`/`runId` and path ids; reject
  whitespace-only content on create; normalize refresh `mode`
