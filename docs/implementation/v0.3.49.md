# v0.3.49 Implementation

## Summary

PLAN_FOR_V0_3_0 residual polish: **Escape preventDefault** on list pages and
Settings (complete hygiene from v0.3.48), **MemoryModal** Escape stack safety,
**Discord/Slack runtime content length** enforcement (Tasks 7–8 messaging),
and **RunDetailPanel Escape test** mocks.

기준: v0.3.48

---

## Changes

### Task 1: Escape preventDefault (list UX)

When Escape is handled, call `preventDefault()` so stacked listeners do not
also act. Updated:

- Blocks, Deployments, DesignSystems, Harnesses, Media, Memory (page),
  ModeSelection, Plugins, Routines, Sessions (page), Skills, Templates,
  Workflows
- Settings MCP OAuth / add-server form
- WorkflowEditor inline name edit (Escape cancel + Enter commit)

### Task 2: MemoryModal Escape hygiene

- **MemoryModal**: respect `defaultPrevented` and call `preventDefault` before
  closing (matches NewSessionModal / ConfirmLeaveModal)

### Task 3: Discord / Slack runtime length (Tasks 7–8)

- Shared constants on `message-text.ts`:
  - `DISCORD_CONTENT_MAX_LENGTH` = 2000
  - `SLACK_CONTENT_MAX_LENGTH` = 4000
- **DiscordMessageNode** / **SlackMessageNode**: reject over-limit content
  before calling the API (aligned with graph validation warnings)

### Task 4: Unit tests

- Discord / Slack over-limit and at-limit cases
- message-text limit exports
- RunDetailPanel Escape tests mock `getWorkflowRun` so mount load does not throw
