# v0.3.0 Implementation

## Summary

Open Design 완전 마이그레이션 + neos-work 자체 기능 공백 해소.
Design Context Layer, Automation Routine, 외부 CLI Spawn + 스트리밍, Live Artifact Preview,
Plugin / Atom Pipeline (MVP), Generative UI / Human-in-the-loop,
Media Generation 노드 (DALL-E 3 / TTS), Deploy 노드 (Vercel / Cloudflare),
MCP OAuth 2.0 (PKCE), ZIP 내보내기/가져오기, 병렬 브랜치 & OR 게이트, 코딩 도메인 내장 블록,
Webhook 트리거, 노드 자동 레이아웃 (dagre), 워크플로우 버전 이력을 포함한다.

- **`apps/server/src/lib/plugin-runner.ts`** (신규)
  - SSE 기반 atom pipeline 실행 엔진
  - stage 순서 실행, `humanInLoop: true` stage는 `stage.waiting` SSE 이벤트 발행 후 Promise 일시 중단
  - In-memory Map (`pendingRuns`)으로 suspend/resume 관리
  - stage 프롬프트의 `{{key}}` 플레이스홀더를 이전 stage 출력으로 보간
  - Anthropic Messages API 우선, OPENAI_API_KEY 폴백

  GET  /api/plugins

  POST /api/plugins/:id/run/:runId/resume

    - 완료/실패 메시지 표시

### Task 9: MCP OAuth 2.0 (OD §14.6)

**개념**: MCP 서버 설정 화면에서 "OAuth" 버튼으로 OAuth 2.0 PKCE 플로우를 시작. 시스템 브라우저에서 인증 완료 후 토큰을 `~/.config/neos-work/mcp-tokens/<serverId>.json`에 저장. CLI 에이전트 spawn 시 토큰을 환경변수로 자동 주입.

- **`apps/server/src/lib/mcp-oauth-store.ts`** (신규)
  - `McpOAuthToken` 인터페이스: `serverId`, `accessToken`, `refreshToken?`, `expiresAt?`, `scope?`, `tokenType?`
  - `saveToken()`, `loadToken()`, `deleteToken()` — `~/.config/neos-work/mcp-tokens/<serverId>.json` CRUD
  - `getTokenStatus(serverId)` — 연결 상태 + 만료 여부 반환 (민감한 토큰 값 제외)
  - `isTokenValid(serverId)` — 만료 검사
  - serverId sanitization으로 path traversal 방어 (`[^a-zA-Z0-9_-]` 제거)

- **`apps/server/src/routes/mcp.ts`** (수정)
  - PKCE 헬퍼: `generateCodeVerifier()` (32바이트 base64url), `generateCodeChallenge()` (SHA-256 S256), `generateState()` (32바이트 hex)
  - In-memory `Map<state, PendingFlow>` — 10분 TTL 자동 만료
  - **`POST /api/mcp-servers/oauth/start`** — PKCE 파라미터 생성, authUrl 조립 후 반환
  - **`GET /api/mcp-servers/oauth/callback`** — state 검증 (`timingSafeEqual`), 토큰 교환, 토큰 파일 저장, 브라우저 close 안내 HTML 반환
  - **`GET /api/mcp-servers/oauth/:serverId/status`** — 연결 상태 JSON 반환
  - **`POST /api/mcp-servers/oauth/:serverId/refresh`** — refresh_token으로 토큰 갱신, 파일 업데이트
  - **`DELETE /api/mcp-servers/oauth/:serverId`** — 토큰 파일 삭제 (revoke)

- **`apps/server/src/lib/cli-agents.ts`** (수정)
  - `loadMcpTokenEnvVars()` — `~/.config/neos-work/mcp-tokens/` 스캔, 유효 토큰을 `NEOS_MCP_TOKEN_<SERVERID_UPPER>` 환경변수 맵으로 반환
  - `spawnCliAgent()` — `env: { ...process.env, ...mcpTokenEnv }` 로 CLI 프로세스에 토큰 주입

- **`apps/desktop/src/pages/Settings.tsx`** (수정)
  - MCP 서버 카드에 OAuth 상태 배지 추가: `● OAuth` (초록) / `○ OAuth` (회색)
  - 카드 로드 시 `GET /api/mcp-servers/oauth/:id/status` 병렬 호출
  - "OAuth" 버튼 → OAuth 설정 모달 (authorizationEndpoint, tokenEndpoint, clientId, scope 입력)
  - "Open Browser" → `startMcpOAuth()` → Tauri `shell.open(authUrl)` 로 시스템 브라우저 오픈, 3초 후 상태 자동 갱신
  - "Disconnect" 버튼 → `revokeMcpOAuth()` 호출

- **`apps/desktop/src/lib/engine.ts`** (수정)
  - `startMcpOAuth(params)` — `POST /api/mcp-servers/oauth/start`
  - `getMcpOAuthStatus(serverId)` — `GET /api/mcp-servers/oauth/:id/status`
  - `revokeMcpOAuth(serverId)` — `DELETE /api/mcp-servers/oauth/:id`
  - `refreshMcpOAuth(serverId, params)` — `POST /api/mcp-servers/oauth/:id/refresh`
