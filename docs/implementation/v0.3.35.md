# v0.3.35 Implementation

## Summary

PLAN_FOR_V0_3_0 residual UX polish: **Deployments status/provider filter prefs**
(Task 8), **Escape** on Workflows create modal, Design Systems create form,
Settings MCP add form, ConfirmLeaveModal, and RunInputsDialog, plus

## Changes

### Task 1: Deployments filter prefs (PLAN Task 8)

- **`deployment-filter-prefs.ts`**: `load` / `save` for status and provider chips
- Shared chip lists: `DEPLOYMENT_STATUS_FILTERS`, `DEPLOYMENT_PROVIDER_FILTERS`
- localStorage keys: `neos-deployments-status`, `neos-deployments-provider`
- Deployments page restores last status / provider selection on mount

### Task 2: Modal Escape (UX)

- **Workflows** new-workflow modal: Escape closes and clears draft name/creating state
- **Design Systems** create form: Escape cancels and clears name/description/error
- **Settings** MCP add-server form: Escape dismisses (OAuth modal still preferred when open)
- **ConfirmLeaveModal**: Escape = Stay (`onCancel`)
- **RunInputsDialog**: Escape cancels run-with-inputs

### Task 3: Graph validation

- **`duplicate_node_label`** warning when two or more nodes share the same
  non-empty label (trim + case-insensitive)
- Blank labels still only emit `missing_node_label` (no duplicate noise)

### Task 4: Unit tests

- `deployment-filter-prefs.test.ts` — chip options, defaults, round-trip, invalid
- `ConfirmLeaveModal.test.tsx` / `RunInputsDialog.test.tsx` — Escape handlers
- `WorkflowValidation.test.ts` — duplicate_node_label cases
