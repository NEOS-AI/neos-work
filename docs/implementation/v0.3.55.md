## Changes

### Task 1: MCP servers + OAuth (Task 9)

- **POST /api/mcp-servers**: trim `name` / `command` / `url`; reject blank name
  or whitespace-only command/url; http transport requires `http(s)` URL
- **POST /api/mcp/oauth/start**: trim all body fields; reject blank required
  fields; `authorizationEndpoint`, `tokenEndpoint`, and `redirectUri` must be
  safe `http(s)` URLs (`isSafeHttpBaseUrl`)

### Task 2: Harness routes (Task 3)

- Create: trim `name`, `systemPrompt`, `domain`, `description`; reject blank
  name/systemPrompt
- Update: same trim rules; blank name/systemPrompt → 400

### Task 3: Preflight deploy project (Task 8)

- **`invalid_deploy_project`** when non-empty `projectName` fails
  `isValidDeployProjectName` (blank still falls back to `neos-deploy` at runtime)

### Task 4: Media / Plugins

- **POST /api/media/generate**: normalize `surface` (trim + lower-case)
- **Plugin upgrade-from-skill**: trim `skillId`, `skillDirName`, `name`,
  `description`

### Task 5: Unit tests

- MCP create trim/reject cases + OAuth start http(s) validation
- Harness create/update trim + blank name
- Preflight invalid vs blank deploy projectName

### Review fixes (post-ship)

- **AgentNode**: clamp harness `maxSteps` to 1–200 (was unbounded when harness set);
  trim blank entries from harness `allowedTools`
- **Harness routes**: clamp constraints (`maxSteps` ≤ 200); trim tool names
- **MCP create**: trim stdio `args` entries; drop blanks
- **Plugins**: trim `:id` / `runId` / `stageId` path/body fields
- **Media generate**: coverage for surface case normalize
- **Blocks create**: trim id/name/prompt/category/domain; validate id charset
- **Memory create/update**: trim name/content; normalize type allow-list
- **Session create**: trim workspaceId/provider/model/title/thinkingMode
- **Routines crystallize**: trim name/description for skill slug
- **Design Systems content PUT**: reject whitespace-only content
