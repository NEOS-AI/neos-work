# v0.3.27 Implementation

## Summary

PLAN_FOR_V0_3_0 residual UX polish: **layout direction prefs** (Task 15),
**relative timestamps** on Dashboard/Routines, **Routines sort**, **RevisionPanel**
delete confirm + count (Task 16), and **self_loop** graph validation.

기준: v0.3.26

---

## Changes

### Task 1: Layout direction persistence (PLAN Task 15)

- **`loadLayoutDirection` / `saveLayoutDirection`** in `layout-prefs.ts`
- localStorage key: `neos-layout-direction` (`TB` | `LR`)
- WorkflowEditor initializes from prefs; Auto Layout always
  `setLayoutDirection(dir)` + `saveLayoutDirection(dir)`
- Fixed missing semicolon on `layout.js` import

### Task 2: Relative time formatting

- **`formatRelativeTime(iso, nowMs?)`** for past/future relative strings
  (`just now`, `30m ago`, `in 6h`, …)
- Dashboard recent workflows: `updatedAt` relative with absolute `title`
- Dashboard recent routines: `nextRunAt` / `updatedAt` relative with absolute `title`

### Task 3: Routines list sort

- Visible routines sorted with **`sortByDateDesc`** by `updatedAt`

### Task 4: RevisionPanel polish (PLAN Task 16)

- Confirm dialog before permanent revision delete
- Header shows revision count `(N)` when loaded

### Task 5: Self-loop validation

- **`self_loop`** error when `edge.source === edge.target`
- Unit coverage in `WorkflowValidation.test.ts`

### Task 6: Unit tests

- `layout-prefs.test.ts` — default TB, round-trip, invalid value
- `format-relative-time.test.ts` — empty/invalid, past, future
- RevisionPanel tests — count badge + delete confirm / cancel
