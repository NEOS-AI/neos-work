> **Version**: 0.1.0
> **Date**: 2026-02-23
> **License**: MIT

## Overview

NEOS Work v0.1.0 is the foundational release of an open-source alternative to Claude Cowork. This version establishes the core architecture — a desktop-first AI chat application with a client-server model, multi-provider LLM support, and an initial tool execution framework.

### What v0.1.0 Delivers

- Tauri v2 desktop app with Host/Client mode
- Real-time AI chat with SSE streaming
- Multi-provider LLM support (Anthropic Claude, Google Gemini)
- Tool execution loop with filesystem tools
- Session, workspace, and settings management
- Dark/Light/System theme support
- Korean/English internationalization

### What v0.1.0 Does NOT Include

- Agent orchestrator / task planning
- Browser automation (Playwright)
- MCP client
- Skill system (parser, discovery, installation)
- CI/CD pipeline
- Security hardening (keychain, auth middleware)

### Client-Server Separation

```
┌─────────────────────────────────────────────────┐
│              Desktop App (Tauri v2)              │
│  ┌──────────────────┐  ┌─────────────────────┐  │
│  │  Frontend (React) │  │  Engine Server      │  │
│  │  - Chat UI        │◄─┤  (Hono, :57286)    │  │

│  │  - Dashboard      │  │  - SQLite DB        │  │
│  └──────────────────┘  └─────────────────────┘  │
│         ▲ HTTP/SSE                               │
│         ▼                                        │

│  └──────────────────┘                            │
└─────────────────────────────────────────────────┘
```

**Host Mode**: Tauri starts the engine server as a sidecar process. The engine binds to `127.0.0.1:57286`.

**Client Mode**: The UI connects to a user-specified remote engine server address.

## Database Schema

v0.1.0 uses 4 tables. The planned `agent_step` and `skill` tables are deferred.

```sql
CREATE TABLE workspace (
  id          TEXT PRIMARY KEY,
  name        TEXT NOT NULL,
  path        TEXT,
  type        TEXT NOT NULL DEFAULT 'local',
  created_at  TEXT DEFAULT (datetime('now')),
  updated_at  TEXT DEFAULT (datetime('now'))
);

CREATE TABLE session (
  id            TEXT PRIMARY KEY,
  workspace_id  TEXT NOT NULL REFERENCES workspace(id) ON DELETE CASCADE,
  title         TEXT,
  provider      TEXT DEFAULT 'anthropic',
  model         TEXT DEFAULT 'claude-sonnet-4-5-20250929',
  thinking_mode TEXT DEFAULT 'none',
  created_at    TEXT DEFAULT (datetime('now')),
  updated_at    TEXT DEFAULT (datetime('now'))
);

CREATE TABLE message (
  id          TEXT PRIMARY KEY,
  session_id  TEXT NOT NULL REFERENCES session(id) ON DELETE CASCADE,
  role        TEXT NOT NULL,     -- 'user' | 'assistant'
  content     TEXT NOT NULL,     -- plain text or JSON (MessageContent[])
  metadata    TEXT,              -- JSON; { structured: true } for tool messages
  created_at  TEXT DEFAULT (datetime('now'))
);

CREATE TABLE setting (
  key         TEXT PRIMARY KEY,
  value       TEXT NOT NULL,
  updated_at  TEXT DEFAULT (datetime('now'))
);
```

**Indexes** (added via code review):
- `idx_session_workspace_id` on `session(workspace_id)`
- `idx_session_updated_at` on `session(updated_at)`
- `idx_message_session_id` on `message(session_id)`
- `idx_message_created_at` on `message(created_at)`

**Notes**:
- WAL journal mode enabled for concurrent read performance
- Foreign keys enforced (`PRAGMA foreign_keys = ON`)
- Default workspace "Starter" auto-seeded on first launch
- DB stored at `~/.neos-work/data.db`

### Settings

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/settings` | Get all settings |
| `GET` | `/api/settings/:key` | Get single setting |
| `PUT` | `/api/settings/:key` | Upsert setting (key validated: `/^[a-zA-Z0-9_.-]+$/`, max 100 chars; value max 1MB) |
| `DELETE` | `/api/settings/:key` | Delete setting |
| `POST` | `/api/settings/verify-key` | Verify API key against provider |

### SSE Streaming (Chat)

The chat endpoint (`POST /api/session/:id/chat`) returns an SSE stream with the following event types:

| Event | Description |
|-------|-------------|
| `text` | AI response text chunk |
| `thinking` | LLM thinking/reasoning chunk |
| `tool_use` | Tool invocation (name, input, toolUseId) |
| `tool_result` | Tool execution result (toolUseId, content) |
| `error` | Error message |
| `done` | Stream completion signal |

**Deviation from plan**: The initial plan specified a `GET /api/session/:id/events` SSE subscription endpoint with granular event types (`message.chunk`, `agent.step`, etc.). v0.1.0 uses a simpler `POST /api/session/:id/chat` pattern where SSE is returned directly from the chat request.

## LLM Provider System

### Architecture

```
LLMProviderAdapter (interface)
├── AnthropicAdapter    → Anthropic SDK
└── GoogleAdapter       → Google AI SDK
         ↓
   ProviderRegistry     → manages adapters, dispatches chat()
```

### Provider Interface

```typescript
interface LLMProviderAdapter {
  id: string;
  name: string;
  models: Model[];
  chat(params: ChatParams): AsyncIterable<ChatChunk>;
  validateApiKey(apiKey: string): Promise<boolean>;
}
```

### Supported Models

| Provider | Model ID | Display Name |
|----------|----------|-------------|
| Anthropic | `claude-opus-4-6` | Claude Opus 4.6 |
| Anthropic | `claude-sonnet-4-5-20250929` | Claude Sonnet 4.5 |
| Anthropic | `claude-haiku-4-5-20251001` | Claude Haiku 4.5 |
| Google | `gemini-2.0-flash` | Gemini 2.0 Flash |
| Google | `gemini-2.0-pro` | Gemini 2.0 Pro |

Model definitions are centralized in `packages/shared/src/models.ts` as a single source of truth used by adapters, server, and desktop UI.

### Thinking Mode

Extended thinking is supported for both providers with configurable budget levels:

| Mode | Token Budget |
|------|-------------|
| `none` | 0 (disabled) |
| `low` | 1,024 |
| `medium` | 4,096 |
| `high` | 16,384 |

### Tool Support

Both adapters translate the unified `ToolDefinition` format into provider-specific tool schemas:

- **Anthropic**: Maps to `tools[].input_schema`, streams `content_block_start/delta/stop` for tool_use blocks
- **Google**: Maps to `functionDeclarations`, parses `part.functionCall` from stream responses. Generates `crypto.randomUUID()` for tool IDs since Google API doesn't provide them

### API Key Flow

```
Settings UI → saveSetting('apiKey.anthropic', key)
           → PUT /api/settings/apiKey.anthropic → SQLite

Chat time:  getRegistry()
           → check request header 'x-anthropic-key'
           → fallback to DB: getSetting('apiKey.anthropic')
           → create adapter with key
```

**Known limitation**: API keys are stored as plain text in SQLite. Tauri keychain integration is planned but not implemented in v0.1.0.

## Tool Execution Framework

### Components

| File | Description |
|------|-------------|
| `packages/core/src/tools/base.ts` | `Tool` interface (`name`, `description`, `inputSchema`, `execute()`) and `ToolResult` interface |
| `packages/core/src/tools/registry.ts` | `ToolRegistry` class — register, get, list, execute tools + convert to LLM-compatible definitions |
| `packages/core/src/tools/filesystem.ts` | Factory functions for filesystem tools |

### Filesystem Tools (v0.1.0)

| Tool | Function | Description |
|------|----------|-------------|
| `read_file` | `createReadFileTool(root)` | Read file contents within workspace |
| `write_file` | `createWriteFileTool(root)` | Write/create files within workspace |
| `list_directory` | `createListDirectoryTool(root)` | List directory entries (dotfiles excluded) |

### Path Security

All filesystem tools use `safePath(workspaceRoot, userPath)` to prevent path traversal:

```typescript
const absoluteRoot = resolve(workspaceRoot);
const resolved = resolve(absoluteRoot, userPath);
const rel = relative(absoluteRoot, resolved);
if (rel.startsWith('..') || (!resolved.startsWith(absoluteRoot + '/') && resolved !== absoluteRoot)) {
  throw new Error('Access denied: path outside workspace');
}
```

This was initially buggy (Critical fix in code review) — the original condition allowed absolute path bypass.

### Tool Execution Loop

The server implements a `while` loop that interleaves LLM calls with tool execution:

```
User message → save to DB
↓
while (iteration < MAX_TOOL_ITERATIONS):    // MAX_TOOL_ITERATIONS = 10
  LLM call (messages + tool definitions) → SSE stream to client
  ├─ tool_use detected:
  │   → execute tool (30s timeout via Promise.race)
  │   → SSE: tool_result event
  │   → append tool_use + tool_result to message history
  │   → continue loop
  └─ text only:
      → save assistant message to DB
      → break
```

**Safety mechanisms** (added via code review):
- Client disconnect detection via `c.req.raw.signal` abort event
- `safeSend()` wrapper that catches write failures
- Per-tool 30-second timeout
- `clientDisconnected` flag checked at loop top, inside stream iteration, and before tool execution

### Structured Message Storage

Tool messages are stored as `MessageContent[]` JSON arrays in the `content` column, with `metadata: '{"structured":true}'` to distinguish them from plain text messages. On load, structured messages are parsed with `try/catch` fallback to plain text.

### Key Hooks

**`useEngine`** (`apps/desktop/src/hooks/useEngine.tsx`)
- Manages engine server connection state (`idle` → `connecting` → `connected` / `error`)
- In Host mode: calls `startEngine()` → sidecar launch → health check
- Health check with exponential backoff: 5s → 10s → 20s → 40s → max 60s
- Auto-recovery from `error` → `connected` on successful health check
- Loads API keys from server on successful connection

**`useTheme`** (`apps/desktop/src/hooks/useTheme.tsx`)
- CSS variable-based theme system with `data-theme` attribute on `<html>`
- Supports `dark`, `light`, `system` modes
- `system` mode listens to `prefers-color-scheme` media query
- Persists to `localStorage` with validation against allowed values

### Chat UI Features

- **Markdown rendering**: `react-markdown` + `remark-gfm` + `rehype-highlight` with syntax-highlighted code blocks and copy button
- **Thinking display**: Collapsible `ThinkingBlock` component — auto-expands during streaming, auto-collapses on completion
- **Tool step cards**: `ToolStepCard` with status icons (spinner/check/X), collapsible input/output JSON display
- **SSE streaming**: Real-time text/thinking/tool events rendered incrementally

### Tauri Sidecar Integration

```
Host Mode selected
→ startEngine() (apps/desktop/src/lib/tauri.ts)
  → Tauri Command::new_sidecar("neos-engine")
  → spawns engine server process
  → returns child PID

Window close event
→ stop_engine() Tauri command
  → kills child process by PID
```

Configuration:
- `tauri.conf.json`: `externalBin: ["binaries/neos-engine"]`
- `capabilities/default.json`: `shell:allow-spawn`, `shell:allow-kill`
- `lib.rs`: `EngineState` struct holds `Option<CommandChild>`, `start_engine`/`stop_engine` commands

### Theme System

CSS variables defined at three levels in `globals.css`:

```css
:root { /* dark by default */ }
[data-theme="dark"] { /* explicit dark */ }
[data-theme="light"] { /* light palette */ }
```

10+ CSS variables including `--bg-primary`, `--bg-secondary`, `--bg-tertiary`, `--text-primary`, `--text-secondary`, `--text-muted`, `--border-color`, `--accent-primary`, etc.

All hardcoded Tailwind color classes were replaced with CSS variable references during implementation and code review.

## Code Review Fixes (Post-Implementation)

10 issues found and fixed across 11 files. Documented in [things_solved.md](../things_solved.md).

### Critical (2)

1. **Path traversal vulnerability** — `safePath()` logic flaw allowed absolute path bypass. Rewritten with `startsWith(absoluteRoot + '/')` guard.
2. **Tool loop safety** — No client disconnect detection or per-tool timeout. Added `AbortSignal` listener, `safeSend()` wrapper, 30s tool timeout via `Promise.race()`.

### High (3)

3. **Settings input validation** — No key/value validation on PUT endpoint. Added regex, length limits.
4. **JSON parse error handling** — Structured message parse failure crashed the chat. Added `try/catch` fallback. Anthropic adapter preserves raw JSON on parse failure as `{ _raw: ... }`.
5. **Model list duplication** — Models hardcoded in 3 places (UI, adapters). Consolidated to `packages/shared/src/models.ts`.

### Medium (3)

6. **Workspace path validation** — No restriction on workspace paths. Added `homedir()` prefix check.
7. **Health check backoff** — Fixed 5s interval replaced with exponential backoff (5s → 60s max). Added auto-recovery.
8. **DB indexes** — 4 indexes added for foreign key and timestamp columns.

### Low (2)

9. **Theme initial value** — localStorage theme validated against allowed values.
10. **Magic number** — Session title truncation `60` extracted to `MAX_TITLE_LENGTH` constant.
