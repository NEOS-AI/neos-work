# v0.2.3 Implementation

## Summary
워크플로우 실행 관측성(Observability) 및 관리 UX 개선을 위한 패치 릴리스.
노드 실행 시간 추적, 워크플로우 복제·내보내기·가져오기, 에디터 미저장 변경 감지, 실행 로그 UX 개선을 포함한다.

## Changes

### Task 1: Node 실행 시간(durationMs) 추적
- **`packages/shared/src/types/api.ts`** — `node.completed` SSE 이벤트 타입에 `durationMs: number` 필드 추가
- **`packages/workflow-engine/src/executor.ts`** — 노드 실행 전후 `Date.now()` 측정; `onEvent` 호출 시 `durationMs: result.durationMs` 포함
- **`packages/workflow-engine/src/executor.test.ts`** — `'node.completed 이벤트에 durationMs를 포함한다'` 테스트 케이스 추가 (typeof number && >= 0 검증)
- **`apps/server/src/routes/workflow.ts`** — `nodeResults` 저장 시 `durationMs: event.durationMs` 포함

### Task 2: 워크플로우 복제(Duplicate)
- **`apps/server/src/db/workflows.ts`** — `duplicateWorkflow(id)` 함수 추가. 원본 조회 후 이름에 ` (copy)` 접미사를 붙여 `createWorkflow()` 호출
- **`apps/server/src/routes/workflow.ts`** — `POST /:id/duplicate` 엔드포인트 추가 (route 충돌 방지를 위해 `/import` 다음에 등록)
- **`apps/desktop/src/lib/engine.ts`** — `duplicateWorkflow(id)` 메서드 추가
- **`apps/desktop/src/pages/Workflows.tsx`** — 각 워크플로우 카드에 ⧉ 복제 버튼 추가; `handleDuplicate(id)` 핸들러 구현

### Task 3: 워크플로우 Export / Import
- **`apps/server/src/routes/workflow.ts`**
  - `GET /:id/export` — `{ name, nodes, edges }` JSON 반환
  - `POST /import` — body의 `{ name, nodes, edges }`로 신규 워크플로우 생성 (route 등록 순서: `/import` → `/:id/duplicate`)
- **`apps/desktop/src/lib/engine.ts`**
  - `exportWorkflow(id, workflowName)` — Blob + `<a download>` 패턴으로 `.json` 파일 다운로드
  - `importWorkflow(data)` — JSON 객체를 서버에 POST하여 워크플로우 생성
- **`apps/desktop/src/pages/Workflows.tsx`**
  - 헤더에 Import 버튼 추가 (숨겨진 `<input type="file">` + `useRef` 트리거)
  - `handleImportFile(e)` — FileReader로 JSON 파싱 → `client.importWorkflow()` → 새 워크플로우로 navigate

### Task 4: 에디터 미저장 변경 감지(Dirty Flag)
- **`apps/desktop/src/pages/WorkflowEditor.tsx`**
  - `savedDraft` 상태 추가; 워크플로우 로드(`loadWorkflow`) 시 초기화
  - `handleSave` / `handleRun` 자동저장 후 `setSavedDraft(draft)` 갱신
  - `isDirty` useMemo — `JSON.stringify(draft) !== JSON.stringify(savedDraft)` 비교
  - `beforeunload` useEffect — `isDirty`가 true일 때만 이탈 경고 핸들러 등록/해제
  - 툴바 워크플로우 이름 오른쪽에 노란 `•` dot 표시 (미저장 상태 시각화)
  - Export 버튼 툴바 추가 (`client.exportWorkflow()` 호출)

### Task 5: 실행 로그(Run Log) UX 개선
- **`apps/desktop/src/pages/WorkflowEditor.tsx`**
  - `nodeLabelMap` useMemo — 노드 id → 레이블 맵 생성; Run Log에서 UUID 대신 레이블 표시
  - `expandedRunLogIdx` 상태 추가 — 클릭 토글 방식 확장
  - `node.completed` 이벤트에 output이 있으면 `▸` 인디케이터 표시; 클릭 시 JSON 미리보기(400자) 인라인 렌더링
  - `handleRun` 호출 시 `setExpandedRunLogIdx(null)` 초기화

### Task 6: i18n 문구 추가
- **`packages/ui/src/i18n/locales/en/common.json`** — `workflow.duplicate`, `workflow.export`, `workflow.import` 추가
- **`packages/ui/src/i18n/locales/ko/common.json`** — 동일 키에 한국어 번역(`복제`, `내보내기`, `가져오기`) 추가
