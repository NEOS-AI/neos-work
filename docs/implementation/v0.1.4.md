## 발견 및 수정된 불일치

### §1. `Sessions.tsx` — reflect 전략 텍스트 불일치

**파일:** `apps/desktop/src/pages/Sessions.tsx`

**문제:** `step_healing` 이벤트 처리 시 `strategy: 'reflect'` 일 때 표시 텍스트가 `'대안 탐색 중...'`으로 구현되어 있었으나, 계획 문서(PLAN_V0_1_4 § Task 5 체크리스트 5-3) 및 구현 문서(impl v0.1.3 §2 SSE 이벤트 설명)에서는 `'반성 중...'`으로 명시.

**수정:**
```diff
- chunk.strategy === 'retry' ? '재시도 중...' : '대안 탐색 중...'
+ chunk.strategy === 'retry' ? '재시도 중...' : '반성 중...'
```

---

### §2. `GoogleAdapter.chat()` — AbortSignal 미전달

**파일:** `packages/core/src/llm/google.ts`

**문제:** `ChatParams`에서 `signal`을 구조분해하지 않아 취소 요청이 Google Gemini API 호출에 전혀 전달되지 않았음. 사용자가 채팅을 취소해도 Google 스트림이 계속 실행되는 버그.

**수정:**
1. `const { model, messages, tools, maxTokens, thinkingMode, signal } = params;` — `signal` 추가
2. `generateContentStream` 호출의 `config`에 `abortSignal: signal ?? undefined` 추가

**근거:** Google GenAI SDK `GenerateContentConfig`에 `abortSignal?: AbortSignal` 필드가 정의되어 있음. Anthropic 어댑터는 이미 `signal`을 전달하고 있었으므로 동등한 처리가 필요함.

---

### §3. `orchestrator.ts` — `retryStrategy` dead code

**파일:** `packages/core/src/agent/orchestrator.ts`

**문제:** `retryStrategy` 필드 선언 및 `RetryStrategy` 인스턴스화 코드가 존재했으나, catch 블록에서 실제로는 `retryStrategy.heal()`을 호출하지 않고 직접 `executeStep()`을 재호출하는 방식으로 구현됨. `retryStrategy` 필드와 `RetryStrategy` import가 dead code.

**수정:**
- `RetryStrategy` import 제거
- `retryStrategy` 필드 선언 및 `new RetryStrategy()` 초기화 제거

**비고:** `RetryStrategy` 클래스 자체는 `healing.ts`에 그대로 유지하고 `agent/index.ts`를 통해 외부로 export됨 (사용자 커스텀 healing 전략에서 유용하게 쓰일 수 있음).

## 의도적 미구현 확인

### H-1. `synthesizeResult`에 ContextManager 미적용

계획 문서 PLAN_V0_1_3의 Task 4 수정 대상 목록에서 orchestrator의 `synthesizeResult`에 ContextManager 적용이 **명시적으로 언급되지 않음**. 구현 문서(impl v0.1.3 §3)도 "서버 통합 포인트"로 chat route만 언급. 따라서 의도적 미구현으로 확인.

`synthesizeResult`는 단일 요약 요청이며 이전 대화 히스토리를 직접 전달하지 않으므로 ContextManager 적용 불필요. 현 구현이 올바름.
