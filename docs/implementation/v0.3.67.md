## Changes

### Task 1: KIS API client (Task 12)

- Trim app key/secret; blank credentials rejected
- Trim symbol; blank → `symbol is required` (no fetch)
- Chart period allow-list `D|W|M` (default `D`); count clamp 1–500
- Network `fetch` failures rethrown as clean `KIS network error`
- Missing/blank `access_token` rejected; truncate provider error text

### Task 2: Parallel / OR gates (Task 11)

- `OrGateNode` and `ORGateNode` fail with clear error when no upstream inputs
  (instead of silently returning `undefined`)

### Task 3: BlockNode prompt/skill (Task 12)

- Prompt blocks: reject blank/whitespace `promptTemplate`
- Skill blocks: reject blank/whitespace `skillId`
- Registry: `registerBlockMeta`, `resolveBlock`/`getNativeExecutor`/`listBlocks` trim

### Task 4: Unit tests

- KIS blank credentials/symbol + network failure
- OR gates empty inputs
- BlockNode empty promptTemplate / skillId

### Also from v0.3.67 review

- **Registry**: trim ids on `registerNativeBlock` / `registerBlockMeta`; blank no-op
- **KIS chart**: uppercase/trim period (`w` → `W`)
- **AND gate / ParallelEnd**: fail when no upstream inputs (consistent with OR)

### Task 5: Unit tests (review)

- Registry register/resolve id trim
- KIS lowercase period normalize
- AND / ParallelEnd empty inputs
